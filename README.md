# Codex Refit

A local starship-console maintenance dashboard and macOS menu bar app for large Codex state folders. It opens in Easy Mode with one `Smart Optimize` button and a `Speed Check` benchmark. Hard Mode reveals the manual controls.

Run it in the browser with:

```sh
npm run dev
```

Build and run it as a local macOS app with:

```sh
npm run package:mac
open "release/Codex Refit.app"
```

The app scans local Codex paths, shows the biggest session, media, log, cache, image, worktree, and memory buckets, and keeps risky actions guarded. `Speed Check` records a stable local readiness score using scan time, state DB latency, log DB latency, WAL size, active transcript size, session media/appshot pressure, task-clarity markers, stale threads, oversized transcript counts, and Codex-managed worktree pressure; it also records a live timing score that accounts for current Codex process load, background terminal commands, default model/reasoning task fit, cloud-handoff readiness, model-provider auth health, named speed-profile health, subagent fan-out settings, custom-agent load, MCP startup pressure, shell environment breadth, context budget settings, response verbosity/reasoning-summary shape, effective web-search mode, deprecated web-search toggles, network/sandbox friction, instruction-stack size, custom instruction overrides, command-rule pressure, lifecycle hook load, automatic approval review, memory-injection weight, deprecated custom prompts, skill-catalog metadata pressure, project local-environment setup/action readiness, and file-based managed Codex config conflicts. Hard Mode shows the score drivers, while Smart Optimize shows a conservative projected score lift before you run it. `Codex Doctor` adds a ranked Next Moves queue plus docs-informed guidance for local state, media-heavy active sessions, task clarity in active threads, cloud handoff for heavy/parallel work, live Codex process load, background terminal work, Codex-managed worktrees, model provider/auth setup, model/effort task fit, subagent fan-out, custom agents, MCP startup, shell environment policy, network sandbox, context/tool-output budget, response shape, profile health, instruction stack, instruction overrides, command rules, managed requirements/defaults, lifecycle hooks, approval review, memory context, skill catalog size, auth cache health, model/reasoning settings, Fast Mode, workflow context, trusted project playbooks, local setup actions, and task-size profiles such as Fast Lane, Small Tasks, and Deep Work. You can also run the official `codex doctor --json` check on demand; Refit summarizes the redacted report and offers safe copyable action snippets instead of slowing every scan. In Hard Mode, Doctor also shows durable config advice for shell snapshots, shell environment policy, network access/proxy scope, effective web-search mode, legacy web-search keys, session media pressure, task-clarity pressure, cloud-handoff pressure, worktree pressure, model-provider auth commands/env vars, model/effort task fit, named speed-profile health, response verbosity/reasoning-summary shape, context budget, active AGENTS guidance, custom base/compact prompt overrides, deprecated custom prompts, command-rule files, managed config layers, auth credential storage, the active service tier, approval flow, Goal Mode, memory use/generation, MCP startup health, lifecycle hook load, reusable AGENTS guidance, named profile files, project `AGENTS.md` and local-environment setup/action counts, live helper pressure, background command pressure, concurrency pressure, custom-agent validity, trusted project count, enabled tool surface, skill catalog pressure, subagent depth/thread caps, and a copyable Fix Kit for safe next steps. Destructive deletion actions stay locked until you switch to Hard Mode and enable `Deletes On`. SQLite state changes create backups under the app data directory.

`Smart Optimize` builds a plan from the current scan. `Safe` runs non-destructive cleanup first: move archived transcripts out of active sessions, archive stale thread rows, compact state, prune/checkpoint logs with a backup, clear crash dumps, and clean rebuildable browser caches. `Recover Space` can delete old archived conversations and old Refit backups after the selected age, but only after deletes are explicitly allowed. `Full Pass` can also move older generated-image folders from `~/.codex/generated_images` to `~/.codex/archived_generated_images`; generated images are never deleted by Codex Refit.

## Codex Speed Playbook

Codex Refit checks three kinds of speed pressure:

- Local state: active transcripts, media-heavy appshot/image sessions, archived pointers still sitting in active sessions, SQLite log/WAL size, crash dumps, and rebuildable browser caches.
- Runtime and config: official Doctor findings, CLI/app version drift, live app-server/helper load, background terminal commands, Codex-managed worktree pressure, default model/reasoning task fit, cloud-handoff Git/GitHub branch readiness, model-provider/env/auth-command health, named speed-profile health, subagent depth/thread fan-out, custom-agent model/instruction/MCP load, MCP/plugin startup surface, shell snapshot and command environment policy, response verbosity/reasoning-summary shape, effective web-search mode and deprecated web-search toggles, network/sandbox mode and proxy policy, context/tool-output budget, instruction-stack size/caps, custom instruction/base/compact prompt overrides, command rules, managed requirements/defaults, lifecycle hook load, approval flow and auto-review mode, memory context settings/storage, auth cache metadata, Fast Mode eligibility, model and reasoning effort, and trusted project entries.
- Workflow context: sampled active-thread task shape, empty or missing global/project `AGENTS.md`, oversized or capped durable guidance, deprecated custom prompts, missing `.codex` local setup/actions, large skill catalogs, overly broad concurrent threads, and task/model mismatch.

The app does not silently rewrite Codex config, inspect auth token contents, print model-provider token values, or print transcript contents while checking media and task-shape pressure. It surfaces the next move and copyable snippets so you can decide whether to run cleanup, archive finished visual-debugging threads, close idle Codex threads and terminals, inspect background work with `/ps`, stop obsolete background terminal work with `/stop`, compact or split sprawling active threads, push or hand off GitHub-backed work to cloud, cap subagent fan-out, review custom agents, review MCP servers with `/mcp verbose`, fix model-provider env/auth-command setup, trim spawned-command environment policy, review network/sandbox settings, review context budget and use `/compact`, review active AGENTS guidance, review custom instruction overrides, test command rules with `codex execpolicy check`, review managed requirements/defaults, review lifecycle hooks with `/hooks`, review approval flow, review memories, review installed skills with `/skills`, migrate deprecated custom prompts into skills, disable stale skills with `[[skills.config]]`, refresh login with `codex doctor`/`codex login`, fix terminal/auth/update/MCP findings from the official Doctor report, create a named `speed.config.toml` profile, add a project playbook, switch model/effort for a small task, check `/fast status`, align CLI/app versions, remove stale trusted paths, or add concise AGENTS guidance.
